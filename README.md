Пятая неделя практики: Логирование событий и Агенты

Выполненные задачи:
1. **Собственный модуль:** Вся логика задания упакована в модуль `my.task` (находится в `local/modules/my.task`).
2. **Логирование изменений:**
    - Реализованы обработчики событий `OnAfterIBlockElementAdd` и `OnAfterIBlockElementUpdate`.
    - Информация об изменениях записывается в инфоблок `LOG`.
3. **Автоматическая структура логов:**
    - Для каждого логируемого инфоблока в ИБ `LOG` автоматически создается раздел (папка).
    - Привязка идет по символьному коду инфоблока через `EXTERNAL_ID` раздела.
4. **Рекурсивный путь:**
    - Написана рекурсивная функция, строящая полную цепочку разделов (от корня до элемента).
5. **Агент очистки:**
    - Создан агент `CleanOldLogsAgent();`, который раз в час удаляет все логи, кроме 10 самых новых.

Инструкция по запуску:

1. **Миграция:**
   Выполнить команду в терминале для создания структуры инфоблока LOG:
   - php bin/migrate up

2. **Установка модуля:**
   - Скопировть папку `my.task` в `/local/modules/`.
   - Зайдите в админпанель: Marketplace -> Установленные решения.
   - Найдите «Тренировочный модуль (Логирование)» и нажать Установить.

3. **Регистрация обработчиков:**
   - Скопировать содержимое файла `init.php` в `/local/php_interface/init.php`.

4. **Запуск агента:**
   - Агент зарегистрируется автоматически после добавления в разделе **Настройки -> Настройки продукта -> Агенты**.
   - Для корректной работы в Docker-окружении нужно убедиться, что в `dbconn.php` параметр `BX_CRONTAB_SUPPORT` установлен в `false`.
